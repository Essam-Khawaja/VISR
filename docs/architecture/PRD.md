# Global PRD

## Project Summary
StraighterNoodles is a calm, early-morning student life dashboard that turns a chaotic day into a clear, visual timeline. It combines calendar events, tasks, weather, and personal context into one scrollable daily briefing.

## Problem
Students juggle classes, assignments, meetings, clubs, errands, gym, transit, and personal routines across multiple platforms. On busy mornings they wake up tired and confused, unsure what to bring, where to go first, or whether they have time for lunch. Current calendar apps show events but don't answer practical morning questions like "What do I need to bring?" or "Do I have a free slot?"

## Target User
Busy university students balancing academics, extracurriculars, work, health, errands, and social life. Especially those who are tired in the morning and want to reduce decision fatigue.

## Core User Journey
1. Student wakes up tired and opens StraighterNoodles
2. App shows a calm greeting and "Before You Leave" checklist of items to pack
3. App displays the day as a vertical, color-coded flowchart timeline
4. Student scrolls through connected events, sees where they need to be and when
5. Student can add events, attach items, find free time, and get weather-aware advice

## Must-Have Features
- [x] Vertical flowchart-style timeline dashboard
- [x] Today's events and tasks displayed chronologically
- [x] "Before You Leave" item checklist
- [x] Manual event/task creation and editing
- [x] Item-to-event mapping (default per category + one-time)
- [x] Weather-based reminders and item suggestions
- [x] Free time slot detection
- [x] Calm, morning-friendly UI with color-coded categories
- [x] User settings (city, timezone, category defaults)

## Nice-to-Have Features
- [ ] Calendar import (.ics file upload)
- [ ] Voice briefing (Web Speech API)
- [ ] Notes attached to events
- [ ] End-of-day rescheduling
- [ ] Transit time blocks
- [ ] Lunch/break gap warnings

## Out of Scope
- User authentication / multi-user
- D2L integration
- Email parsing
- AI-powered summaries
- Mobile native app

## Success Criteria
The hackathon demo succeeds if:
- [x] The main user flow works end-to-end
- [x] The product is understandable in under 30 seconds
- [x] The app has a clear wow moment (the morning checklist + timeline)
